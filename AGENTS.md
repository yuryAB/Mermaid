# AGENTS.md

## Instrucoes do projeto

- Nao rode build automaticamente.
- Nao rode testes finais automaticamente.
- Faca as mudancas pedidas e deixe os arquivos prontos para verificacao.
- O usuario fara o build e os testes manualmente.
- Checks leves que nao substituem build/test, como leitura de arquivos, buscas, `git diff --check` e inspecoes estaticas, podem ser usados quando ajudarem a validar a edicao.
- Se uma tarefa realmente exigir build ou teste para diagnostico, explique isso ao usuario e aguarde confirmacao explicita antes de executar.

## Icones e assets

- Antes de adicionar, exportar, renomear, redimensionar, substituir ou organizar icones/assets, leia e siga `docs/ICON_EXPORT_INSTRUCTIONS.md`.
- Antes de trocar um icone existente, inspecione o contexto exato no app e preserve a linguagem visual do fluxo.
- Se o asset atual parecer ambiguo, inconsistente ou afetado por conflito de nome, inspecione os assets locais e o historico git antes de substituir.
- Nao invente nem troque icones sem essa verificacao.
